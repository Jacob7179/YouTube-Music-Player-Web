# YouTube Music Player — Android APK Build Guide

Build the web player as an Android APK with Capacitor 7, native media controls, file import/export, and Android background-playback support.

[Main README](../README.md) · [Report an Issue](https://github.com/Jacob7179/YouTube-Music-Player-Web/issues)

</div>

> [!IMPORTANT]
> The included automatic build is designed for **64-bit Windows**. The first build requires an internet connection because portable Node.js, Java, Android SDK components, Gradle dependencies, and npm packages may need to be downloaded.

> [!NOTE]
> A system-wide Java JDK or Android SDK is not required. The build can keep its tools inside `android_build` so the project remains portable between Windows computers.

## Quick Start

From the repository root, run:

```powershell
cd android_build
.\build.bat
```

After a successful build, the APK is available at:

```text
android_build/android/app/build/outputs/apk/debug/app-debug.apk
```

The output is a **debug APK** intended for development and testing. Publishing through Google Play requires a signed release build, which is not configured by this script.

## Build Environment

The current project uses the following main components:

| Component | Project configuration |
|---|---|
| Node.js and npm | System installation when available; otherwise portable Node.js LTS |
| Java | Eclipse Temurin JDK 21 |
| Android platform | Android SDK Platform 35 |
| Android build tools | 35.0.0 |
| Capacitor | 7 |
| Filesystem | `@capacitor/filesystem` 7 |
| Share | `@capacitor/share` 7 |
| File picker | `@capawesome/capacitor-file-picker` 7.2.0 |
| Media session | `@capgo/capacitor-media-session` 7.3.0 |

Dependency versions are locked by `package-lock.json`. Keep this file in Git and prefer `npm ci` when restoring the existing dependency set.

## Project Structure

```text
android_build/
├─ android/                 Generated native Android project
├─ android-sdk/             Portable Android SDK
├─ tools/
│  ├─ jdk-21/              Portable Java JDK 21
│  └─ nodejs/              Portable Node.js when required
├─ www/                     Web files packaged inside the APK
├─ image_source/
│  └─ res/                  Source icon and splash resources
├─ scripts/
│  ├─ configure-android-background-playback.js
│  └─ patch-media-session-plugin.js
├─ build.bat                Complete automatic APK build
├─ clean.bat                Generated-folder cleanup
├─ capacitor.config.json    Capacitor app configuration
├─ package.json             npm scripts and dependencies
└─ package-lock.json        Locked dependency versions
```

The `android` folder is generated by Capacitor. Native files changed directly inside it may be replaced by a later synchronization or regeneration. Keep repeatable native modifications in the scripts under `android_build/scripts`.

## What `build.bat` Does

The complete build performs these steps:

1. Uses the available system Node.js/npm or downloads portable Node.js LTS.
2. Restores npm dependencies from `package-lock.json` when required.
3. Applies the media-session plugin patch.
4. Downloads and selects portable Java JDK 21 when missing.
5. Downloads Android SDK Platform 35, Build Tools 35.0.0, and Platform Tools when missing.
6. Creates the Capacitor Android project when it does not exist.
7. Writes `android/local.properties` with the selected SDK path.
8. Synchronizes the web application and native Capacitor plugins.
9. Reapplies Android background-playback changes after synchronization.
10. Adds the Android foreground media-playback permission.
11. Copies icon and splash resources from `image_source/res`.
12. Stops old Gradle daemons, cleans the Android build, and creates the debug APK.

Use the complete build after changing dependencies, Capacitor configuration, native patches, Android resources, or generated Android files.

## Safe Quick Rebuild for Web-Only Changes

Use this method only after `build.bat` has succeeded at least once and the following folders already exist:

```text
android/
android-sdk/
tools/jdk-21/
```

First update the web files inside `android_build/www`. Then run:

```powershell
cd android_build

$env:JAVA_HOME = (Resolve-Path ".\tools\jdk-21").Path
$env:PATH = "$env:JAVA_HOME\bin;$env:PATH"
$env:ANDROID_HOME = (Resolve-Path ".\android-sdk").Path
$env:ANDROID_SDK_ROOT = $env:ANDROID_HOME

npx cap copy android

cd android
.\gradlew.bat assembleDebug
```

`npx cap copy android` copies web assets without performing a full plugin synchronization. This avoids unnecessarily replacing native files that the complete build patches after `cap sync`.

> [!WARNING]
> Do not use the quick method after changing npm dependencies, Capacitor plugins, `capacitor.config.json`, media-session code, Android permissions, icons, or splash resources. Run `build.bat` instead.

## Updating Web Files

Capacitor packages the files stored in:

```text
android_build/www
```

Changes made only to the repository-root `index.html`, `script.js`, or `style.css` are not automatically copied into `android_build/www` by Capacitor. Keep the APK copy synchronized before rebuilding.

The Android version may intentionally contain small differences from the hosted web version, so review changes before replacing the complete `www` folder.

## Install the Debug APK

### Manual installation

Copy this file to the Android device and open it:

```text
android/app/build/outputs/apk/debug/app-debug.apk
```

Android may ask for permission to install applications from the file manager or browser used to open the APK.

### Installation with ADB

Enable **Developer options** and **USB debugging**, connect the device, and run:

```powershell
cd android_build
.\android-sdk\platform-tools\adb.exe devices
.\android-sdk\platform-tools\adb.exe install -r .\android\app\build\outputs\apk\debug\app-debug.apk
```

The `-r` option updates the installed debug app while preserving its application data when the signing key is compatible.

If ADB reports `INSTALL_FAILED_UPDATE_INCOMPATIBLE`, the installed app was signed with a different key. Uninstalling it resolves the conflict but removes its local app data:

```powershell
.\android-sdk\platform-tools\adb.exe uninstall com.jacob7179.youtubemusicplayer
```

Then run the install command again.

## Native Android Features

### Playlist import and export

The APK uses native Capacitor plugins for file selection, storage, and sharing:

```text
@capacitor/filesystem
@capacitor/share
@capawesome/capacitor-file-picker
```

The hosted GitHub Pages and Vercel versions continue using browser-based file import and download behavior.

### System media controls

The application supports:

- Play and pause
- Previous and next
- Seek controls
- Song metadata
- Playback-position updates
- Android notification and lock-screen controls
- Compatible headset and earphone buttons

Browsers use the Web Media Session API where supported. The Capacitor APK uses `@capgo/capacitor-media-session` for native Android integration.

The operating system does not receive the full JavaScript playlist as a universal native queue. Previous and next commands call the application's actual playlist logic.

### Background playback

The complete build applies Android-specific background playback support:

- A custom Capacitor WebView avoids pausing the embedded player only because the app window becomes hidden.
- Capacitor `KeepRunning` remains enabled so JavaScript can continue after pressing Home or locking the screen.
- A media-playback foreground service keeps system media controls available.
- A partial wake lock is held only while playback is active.
- Android 13 and newer can request notification permission for media controls.

After installing a new APK:

1. Open the app and start a song.
2. Grant notification permission when Android requests it.
3. Test playback after pressing Home and after locking the screen.
4. Set the app battery mode to **Unrestricted** when the device manufacturer aggressively stops background apps.

The YouTube player is still an embedded web player rather than a native audio stream. Force-stopping the app will end playback. Swiping the app away may also stop playback on devices that terminate the WebView process.

### Wired-earphone multi-click controls

The patched Android media callback recognizes the center button on compatible wired earphones:

```text
1 tap  = Play or pause
2 taps = Next song
3 taps = Previous song
```

The taps must occur close together. Single- and double-tap actions wait briefly to determine whether another tap follows.

Button behavior varies by Android device, USB-C adapter, headset firmware, and manufacturer media-routing implementation.

## Native Patch Files

The complete build reapplies these scripts after dependency restoration and Capacitor synchronization:

```text
scripts/patch-media-session-plugin.js
scripts/configure-android-background-playback.js
```

`patch-media-session-plugin.js` corrects Android Play/Pause notification actions and adds headset multi-click handling for the current media-session plugin version.

`configure-android-background-playback.js` applies the custom WebView and background-execution changes to the generated Android project.

Do not remove the `postinstall` script from `package.json`. After changing npm dependencies or native plugins, use:

```powershell
cd android_build
npm ci
.\build.bat
```

Running only `npx cap sync android` can replace generated native changes without reapplying every project patch.

## App Icon and Splash Screen

Source resources are stored under:

```text
android_build/image_source/res
```

The full build copies supported drawable and mipmap folders into the generated Android project after Capacitor synchronization.

Common launcher-icon folders include:

```text
mipmap-mdpi
mipmap-hdpi
mipmap-xhdpi
mipmap-xxhdpi
mipmap-xxxhdpi
mipmap-anydpi-v26
```

Splash resources are stored in the corresponding `drawable`, `drawable-port-*`, and `drawable-land-*` folders.

Edit the source resources under `image_source/res`, not only the generated files under `android/app/src/main/res`, because the next complete build may replace generated resources.

## Verify the Java Version

From `android_build/android`, run:

```powershell
.\gradlew.bat --version
```

The output should report Java 21, for example:

```text
JVM: 21
```

When Gradle is using an older Java installation, reset the environment and stop the existing daemon:

```powershell
cd android_build

$env:JAVA_HOME = (Resolve-Path ".\tools\jdk-21").Path
$env:PATH = "$env:JAVA_HOME\bin;$env:PATH"

cd android
.\gradlew.bat --stop
.\gradlew.bat --version
.\gradlew.bat clean
.\gradlew.bat assembleDebug
```

## Troubleshooting

### Java 21 cannot be found

Typical error:

```text
Cannot find a Java installation on your machine matching this task's
requirements: {languageVersion=21}
```

Run `build.bat` again and confirm this file exists:

```text
android_build/tools/jdk-21/bin/java.exe
```

Then stop old Gradle daemons before rebuilding:

```powershell
cd android_build\android
.\gradlew.bat --stop
```

### Android SDK location is missing

Typical error:

```text
SDK location not found
```

Confirm this file exists:

```text
android_build/android/local.properties
```

Its content should point to the Android SDK with escaped Windows backslashes:

```properties
sdk.dir=C:\\path\\to\\android_build\\android-sdk
```

Running `build.bat` recreates this file automatically.

### A Capacitor plugin is missing

Restore the locked dependency set and run the full build:

```powershell
cd android_build
npm ci
.\build.bat
```

### Node.js or npm fails

The script uses a system npm installation when one is found. If that installation is outdated or damaged, it may fail before the portable fallback is used.

Check the selected versions:

```powershell
node --version
npm --version
```

Use a supported Node.js LTS installation or temporarily remove the incompatible Node.js path from the current terminal so `build.bat` can install its portable copy.

### Notification controls do not appear

- Start playback once after installing the APK.
- Allow notification permission on Android 13 or newer.
- Confirm notifications are enabled for the app in Android settings.
- Rebuild with `build.bat` after changing media-session dependencies.

### Playback stops in the background

- Set the app battery mode to **Unrestricted**.
- Allow background activity and notifications where the device provides those settings.
- Do not force-stop the app.
- Some devices terminate embedded WebViews when the app is removed from recent apps; this cannot be fully prevented by the web-based YouTube player.

### The `tools` folder cannot be deleted

A Gradle daemon, Java process, or Node.js process may still be using a portable executable.

```powershell
cd android_build\android
.\gradlew.bat --stop
```

Then close related terminals and run `clean.bat` again.

### `flatDir` warning

This message is normally a warning rather than the reason the build failed:

```text
WARNING: Using flatDir should be avoided because it doesn't support any meta-data formats.
```

Read the final Gradle exception below the warning to find the actual failure.

## Clean Generated Files

Run:

```powershell
cd android_build
.\clean.bat
```

The cleanup script removes generated build folders and asks before deleting `android-sdk`. Keeping the SDK, JDK, and Node.js tool folders makes later builds faster and reduces repeated downloads.

## Internet Requirements and Limitations

The installed APK requires internet access for:

- YouTube playback
- YouTube search
- Lyrics retrieval
- Lyrics translation
- Any externally hosted styles, scripts, or APIs used by the web application

The APK does not provide offline YouTube audio downloads. Its availability and behavior depend on the embedded YouTube player, network access, browser/WebView support, Android background restrictions, and third-party API availability.
